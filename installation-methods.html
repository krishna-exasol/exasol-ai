Exasol AI Installation Methods

A practical comparison of one-command install paths for bundling Exasol Nano, Exasol JSON Tables, and Exasol MCP Server. The MVP path is Docker-only with separate Windows and macOS/Linux bootstrap commands.

Best MVP GitHub Release asset installer for PowerShell and shell, backed by Docker Compose.
Most Reliable Script Mode Download installer, verify SHA256, then execute. Longer command, stronger integrity.
Best Mature Windows Winget package after a signed installer exists and repository validation passes.
Best Mature macOS/Linux Homebrew tap first. Homebrew core later if the project meets acceptance criteria.

Recommended MVP Commands

Windows PowerShell

irm https://raw.githubusercontent.com/krishna-exasol/exasol-ai/main/install.ps1 | iex

macOS / Linux

curl -fsSL https://raw.githubusercontent.com/krishna-exasol/exasol-ai/main/install.sh | sh

Dependency Conflict Resolution

Problem

exasol-json-tables currently requires pyexasol>=2.2,<3, while exasol-mcp-server currently requires pyexasol>=1,<2. A single Python environment cannot satisfy both constraints.

MVP Solution

Run JSON Tables and MCP Server in separate Docker containers. This keeps both dependency sets valid and gives the user one Docker-based install path.

Release Criteria

  • Pin Nano. Use a tested tag or immutable image digest instead of latest.
  • Pin JSON Tables. Use a tested tag or commit instead of main.
  • Pin MCP Server. Use an exact package version such as 1.10.1.
  • Verify assets. Publish SHA256 checksums for installer assets and generated packages.
  • Default to local access. Bind SQL, Web UI, and MCP ports to 127.0.0.1.
  • Keep MCP read-only by default. Write tools should require explicit user action.
